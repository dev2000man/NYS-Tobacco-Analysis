NYS Tobacco and Vapor Vendors - Data Cleaning and Analysis¶

        #### Created by: Peter A. Blackwell      Last Updated: March 2024 (cleaned up March 2026)

The purpose of this analysis is purely a freelance data cleaning and analysis exercise using Python and an available government dataset.

In this case, the source of this dataset is a New York State government health site - in this case, 'Active Retail Tobacco and Vapor Product Vendors'

https://health.data.ny.gov/Health/Active-Retail-Tobacco-and-Vapor-Product-Vendors/9ma3-vsuk/about_data

Note that there is only one date field on the dataset (CREATION DATE) with the same value for all fields - which precludes date-themed graphs, but we can do other types of analyses.

Custom function to get a Zip code using OpenStreetMap

Input - the ZIP, LONGITUDE, and LATITUDE fields from the dataframe

Function logic: If the ZIP is > 10000 (valid New York State ZIP), then assume we are good - return the ZIP value else - use the geolocator reverse package to search for the ZIP based off the LONGITUDE, and LATITUDE - return that value, or an error if any

import geopy
from geopy.geocoders import Nominatim

from geopy.exc import GeocoderTimedOut, GeocoderServiceError

# Initialize geocoder with a descriptive user_agent
geolocator = Nominatim(
    user_agent="NYSTVPANLSYS",
    timeout=10  # Timeout in seconds
)

def get_zipcode(zip_field, lat_field, lon_field):    
    if (zip_field > 10000):
        return (zip_field)
    else:
#        print(lat_field , ' ', lon_field)
        try:
            location = geolocator.reverse("{}, {}".format(lat_field, lon_field))
            return location.raw['address']['postcode']
        except (AttributeError, KeyError, ValueError):
            print(repr(e), lat_field, lon_field)
        return None

from geopy.extra.rate_limiter import RateLimiter

# 1. Create a "Rate Limited" version of your geolocator
# min_delay_seconds=1 ensures you stay within the 1 req/sec policy
geocode_with_delay = RateLimiter(geolocator.reverse, min_delay_seconds=1)

# 2. Modify your function slightly to use the rate limiter
def get_zipcode_safe(row):
    # Skip if ZIP is already valid
    if row['ZIP'] > 10000:
        return row['ZIP']
    
    try:
        # Construct coordinates string
        query = f"{row['LATITUDE']}, {row['LONGITUDE']}"
        location = geocode_with_delay(query)
        
        if location:
            return location.raw['address'].get('postcode')
    except Exception as e:
        print(f"Error at {row['LATITUDE']}: {e}")
        
    return None

# 3. Apply the function row-by-row (axis=1)
# Note: This will take 1 second per missing ZIP code!
df['TEMPZIP'] = df.apply(get_zipcode_safe, axis=1)

Create choropleth map

The JSON data will be retrieved from another master dataset -

https://raw.githubusercontent.com/plotly/datasets/master/geojson-counties-fips.json

In addition, we will use the 'from urllib.request import urlopen' and 'with' statement to get the connection for the json

In [27]:
import json

from urllib.request import urlopen

with urlopen('https://raw.githubusercontent.com/plotly/datasets/master/geojson-counties-fips.json') as response:
    counties = json.load(response)

fig = px.choropleth(df_counties, geojson=counties, locations="FIPS", color="Stores by County",
                    color_continuous_scale='Viridis', 
                    range_color=(0, 256),
                    scope='usa',
                    hover_name = 'County',
                    labels={'count': 'store count'}
                    )

fig.update_layout(
    title={
        'text': '<b>Tobacco Store count by county in New York State</b>',
        'y':0.9,
        'x':0.5,
        'xanchor': 'center',
        'yanchor': 'top'
    }
)

fig.update_geos(fitbounds='locations', visible=False) # use this to isolate the state!!!
fig.update_layout(margin={'r': 0, 't': 0, 'l': 0, 'b': 0})
fig.show()
